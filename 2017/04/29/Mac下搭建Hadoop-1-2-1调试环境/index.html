Mac下搭建Hadoop-1.2.1调试环境

2017-04-30

本机环境

  1. macOS Sierra (version 10.12.4)
  2. Java 1.8.0_121
  3. Hadoop-1.2.1
  4. IntelliJ IDEA (Community Edition)

准备工作

  1. 下载Hadoop-1.2.1源代码。比如这里

  2. 解压: tar -zxvf hadoop-1.2.1.tar.gz

  3. 用IntelliJ导入

    • Import Project

      ​

    • Create project from existing sources

      ​

    • 一路Next点下去

    • 右键build.xml并Add as Ant Build File

    ​

    • 找到src/mapred/org/apache/hadoop/mapreduce/lib/partittion/InputSampler改一行代码

    • Ant clean and compile

    ​

  4. 修改配置文件，搭建伪分布式系统（可参考文章末第二个链接）

  5. 修改/bin/hadoop将最后两个exec改为echo

    ​

Hadoop客户端调试

  1. 运行并记录参数

    • 参数为-Dproc_-jar……/jsp-api-2.1.jar那一堆
    • Intellij中Run/Edit Configurations，然后’+’，选application后填写

    ​

  2. 修改恢复/bin/hadoop文件，在hdfs中按照configuration配置输入和输出文件的路径

  3. 找到RunJar.java在main函数处加断点，调试配置好的configuration

    ​

    ​

Hadoop组件调试

  1. IntelliJ中Edit Configurations，”+”，Remote，并填写端口号。四个组件同理。

    ​

  2. 修改/conf/hadoop-env.sh，端口与configuration中一致

    ​

  3. 分别找到DataNode，NameNode，TaskTracker，JobTracker的main函数，添加断点

  4. 运行Hadoop环境

  5. 运行jps指令，四个组件应都显示”main class information unavailable“。

  6. IntelliJ中分别对四个Configuration进行Debug

    ​

参考

  1. http://www.techbite.in/2013/05/debug-hadoop-source-code-using.html

  2. http://hadoopindepth.blogspot.jp/2014/08/installing-hadoop-121single-node-in.html

  3. https://understandbigdata.wordpress.com/2012/12/17/debug-custom-java-hadoop-code-in-local-environment/
